# 做市模拟小游戏 — 详细需求文档

## 1. 项目概述

### 1.1 项目定位
一款 Web 端多人实时做市模拟游戏，支持最多 10 名玩家同时参与。玩家围绕一个隐藏的真实值（Value）进行报价和交易，通过做市赚取价差利润。游戏结束后根据 PnL 排名决出胜负。

### 1.2 核心玩法
- 系统设定一个隐藏的真实值（Value），并逐轮释放相关信息
- 玩家按顺序轮流报价（Bid / Ask）
- 非报价玩家可选择与当前报价方成交
- 游戏结束后以真实值结算所有仓位，计算最终 PnL

---

## 2. 角色定义

| 角色 | 说明 |
|------|------|
| **管理员（Admin / 裁判）** | 创建游戏房间、设定游戏参数（Value、Tick Size、价差上限、最大持仓等）、控制游戏流程（开始/暂停/结束）、发布信息 |
| **玩家（Player）** | 登录后加入房间，参与报价和交易 |
| **观众（Spectator）**（可选） | 仅观看，不参与交易 |

---

## 3. 游戏参数（由管理员配置）

| 参数 | 符号 | 说明 | 示例 |
|------|------|------|------|
| 真实值 | `V` | 游戏结算价格，对玩家隐藏 | 50.00 |
| 最小价格单位 | `tick_size` | 报价必须是 tick_size 的整数倍 | 0.5 |
| 初始最大价差比例 | `max_spread_pct` | 首次报价的 (ask - bid) / mid 上限（百分比），避免暴露真实值的绝对规模 | 20% |
| 最大净持仓 | `k` | 每人最大净多头/净空头仓位数 | 10 |
| 玩家人数 | `n` | 本局参与玩家数 | 10 |
| 总轮次 | `total_rounds` | 游戏进行的总轮次 | 5 |
| 每轮报价时限 | `quote_timeout` | 每人报价的最长时间（秒） | 30 |
| 每轮成交时限 | `trade_timeout` | 报价后等待成交的最长时间（秒） | 10 |

## 4. 核心游戏规则

### 4.1 初始状态

每位玩家的初始状态：
- **多头仓位**：10 张（默认持有）
- **空头仓位**：10 张（默认持有）
- **净仓位**（net position）：0
- **现金余额**（cash）：0
- **初始 PnL**：0（因为 10 × V + 10 × (-V) = 0）

> 说明："持有 10 张多头 + 10 张空头"等价于初始净仓位为 0，便于理解后续交易对 PnL 的影响。

### 4.2 报价规则

1. **报价顺序**：每轮中，所有玩家按固定序号（1 → 2 → ... → n → 1 → ...）依次报价
2. **报价内容**：报出一个 Bid 和一个 Ask，必须满足：
   - `Bid < Ask`
   - Bid 和 Ask 均为 `tick_size` 的整数倍
   - `Ask - Bid ≤ 当前允许的最大价差`
3. **报价数量**：固定为 1 张（每次报价只挂 1 张买单 + 1 张卖单）
4. **超时处理**：超时未报价视为放弃本次报价权，自动跳过

### 4.3 价差递减规则

1. 每轮第一位报价者的价差上限为 `mid × max_spread_pct`（其中 mid 为报价者自行决定的报价中点）
2. 若上一位报价者的报价**未被成交**，下一位报价者的允许价差必须比上一位的实际价差至少缩小 1 个 `tick_size`
3. 若上一位报价者的报价**被成交**，下一位报价者的允许价差可以**恢复到上一位的实际价差**（不强制缩小）
4. 若允许价差缩小至 `tick_size`（最小可能价差），后续报价者的价差维持在 `tick_size`

### 4.4 成交规则

1. **成交窗口**：一位玩家完成报价后，进入成交窗口（持续 `trade_timeout` 秒）
2. **成交方式**：
   - 其他玩家点击「买入」→ 以报价方的 **Ask** 价格买入 1 张（买方多头 +1，报价方多头 -1）
   - 其他玩家点击「卖出」→ 以报价方的 **Bid** 价格卖出 1 张（卖方空头 +1，报价方空头 -1）
3. **成交限制**：
   - 每个报价的 Bid 侧和 Ask 侧各只能被成交 **1 次**（先到先得）
   - 一位玩家在同一个报价上最多只能成交一侧（不可同时买和卖）
4. **成交优先级**：谁先点击谁先成交（时间优先）
5. **自成交禁止**：报价方不能与自己的报价成交

### 4.5 仓位管理

1. **净仓位计算**：`net_position = 多头仓位 - 空头仓位`（初始为 0）
2. **最大持仓限制**：
   - `net_position` 的绝对值不能超过 `k`
   - 即：`-k ≤ net_position ≤ k`
3. **达到持仓上限时**：
   - 只能进行反方向交易（如净多头达到 k，只能卖出）
   - 无法进行报价（因为报价必须同时挂双边），系统自动跳过该玩家的报价轮次
   - 仍可参与成交（反方向）

### 4.6 PnL 计算

#### 实时 PnL（Mark-to-Market）
```
unrealized_pnl = net_position × (market_mid_price - V_initial_mid)
realized_pnl = 累计已平仓收益
total_pnl = realized_pnl + unrealized_pnl
```

其中 `market_mid_price` 取最近一次报价的 `(Bid + Ask) / 2`。

#### 最终结算 PnL
游戏结束后，所有未平仓位以真实值 V 结算：
```
final_pnl = cash + net_position × V
```

其中 `cash` 为所有交易产生的现金流累计：
- 买入 1 张时：`cash -= 成交价格`
- 卖出 1 张时：`cash += 成交价格`

> 示例：玩家以 Ask=52 卖出 1 张，又以 Bid=48 买入 1 张，此时 cash = 52 - 48 = 4。若 V=50，net_position=0，则 final_pnl = 4。

## 5. 游戏流程

### 5.1 完整流程

```
┌─────────────────────────────────────────────────────┐
│                  管理员创建房间                        │
│      设定 V, tick_size, max_spread_pct, k 等          │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│              玩家登录 → 加入房间                       │
│            等待所有玩家就绪 / 管理员开始                  │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│                 游戏开始（第 1 轮）                     │
│             管理员可选择发布第 1 条信息                   │
└──────────────────────┬──────────────────────────────┘
                       ▼
          ┌────────────────────────┐
          │  玩家 i 进行报价        │ ◄─── 超时/跳过 ───┐
          │  (Bid, Ask)            │                    │
          └───────────┬────────────┘                    │
                      ▼                                 │
          ┌────────────────────────┐                    │
          │  成交窗口（trade_timeout）│                   │
          │  其他玩家可点击买/卖      │                   │
          └───────────┬────────────┘                    │
                      ▼                                 │
          ┌────────────────────────┐                    │
          │  更新所有仓位 & PnL      │                   │
          └───────────┬────────────┘                    │
                      ▼                                 │
          ┌────────────────────────┐                    │
          │  下一位玩家报价?         │ ── 是 ──────────┘
          └───────────┬────────────┘
                      │ 否（本轮结束）
                      ▼
          ┌────────────────────────┐
          │  是否还有下一轮?         │ ── 是 → 发布新信息 → 重置价差 → 新一轮
          └───────────┬────────────┘
                      │ 否
                      ▼
┌─────────────────────────────────────────────────────┐
│                  游戏结算                              │
│          公布 V，计算最终 PnL，排名                     │
└─────────────────────────────────────────────────────┘
```

### 5.2 轮次与信息机制

- 游戏分多轮进行，每轮开始时管理员可发布一条与真实值 V 相关的信息（线索）
- 信息发布后，所有玩家可见
- 随着轮次推进，信息越来越具体，玩家对 V 的估计应越来越精确
- 每轮内所有玩家依次完成一次报价后，进入下一轮

### 5.3 轮次内报价顺序

- 第 1 轮：玩家 1 → 2 → 3 → ... → n
- 第 2 轮：玩家 2 → 3 → ... → n → 1（首位报价者轮换，避免先手优势）
- 以此类推

## 6. 功能模块

### 6.1 用户系统

| 功能 | 说明 |
|------|------|
| 注册/登录 | 支持用户名 + 密码登录（简单模式），或管理员分配房间码直接加入 |
| 个人主页 | 展示历史战绩、总 PnL、参与场次 |
| 头像/昵称 | 可自定义昵称，系统分配默认头像 |

### 6.2 房间管理

| 功能 | 说明 |
|------|------|
| 创建房间 | 管理员设定参数后创建，生成房间码 |
| 加入房间 | 玩家输入房间码加入 |
| 房间大厅 | 显示已加入的玩家列表、就绪状态 |
| 开始游戏 | 管理员确认后开始 |
| 踢出玩家 | 管理员可在游戏开始前踢出玩家 |

### 6.3 游戏面板（核心页面）

#### 6.3.1 公共信息区
- 当前轮次 / 总轮次
- 当前报价者及其序号
- 倒计时（报价时限 / 成交时限）
- 已发布的信息列表（线索）
- 当前允许的最大价差

#### 6.3.2 报价板（Order Book）
- 当前报价者的 Bid / Ask 展示
- 历史报价记录（本轮所有已报价格）
- 成交记录（谁买了谁、价格、时间）

#### 6.3.3 操作区
- **报价模式**（轮到自己时）：
  - Bid 输入框 + Ask 输入框
  - 价差实时校验提示
  - 「提交报价」按钮
- **成交模式**（他人报价时）：
  - 「买入（Buy @ Ask）」按钮
  - 「卖出（Sell @ Bid）」按钮
  - 按钮上实时显示价格
  - Bid 侧/Ask 侧已被成交时按钮置灰

#### 6.3.4 个人仓位区
- 当前净仓位（可视化，如仓位条）
- 当前 cash
- 实时 PnL（Mark-to-Market）
- 持仓上限提醒

#### 6.3.5 排行榜
- 实时 PnL 排名（所有玩家）
- 可折叠/展开

#### 6.3.6 交易历史
- 本局所有个人交易记录
- 包含：时间、方向（买/卖）、价格、对手方

### 6.4 结算页面

| 内容 | 说明 |
|------|------|
| 真实值公布 | 大字展示 V |
| 最终排名 | 按 final_pnl 降序 |
| 个人详情 | 每人的 final_pnl、净仓位、cash、交易次数 |
| 交易回顾 | 可回看每轮报价和成交情况 |

### 6.5 历史记录

| 功能 | 说明 |
|------|------|
| 历史场次列表 | 展示参与过的所有游戏 |
| 历史 PnL 曲线 | 按时间/轮次展示 PnL 变化 |
| 历史排名 | 每局的排名记录 |
| 历史操作 | 每局的详细交易记录 |

## 7. 数据模型

### 7.1 用户（User）
```
{
  id: string,
  username: string,
  password_hash: string,
  nickname: string,
  avatar_url: string,
  created_at: timestamp,
  total_games: number,
  total_pnl: number
}
```

### 7.2 游戏房间（GameRoom）
```
{
  id: string,
  room_code: string,          // 6位房间码
  admin_id: string,           // 管理员用户ID
  status: enum,               // waiting | playing | finished
  config: {
    value: number,            // 真实值 V（仅管理员可见）
    tick_size: number,
    max_spread_pct: number,   // 百分比，如 0.2 表示 20%
    max_position: number,     // k
    player_count: number,     // n
    total_rounds: number,
    quote_timeout: number,    // 秒
    trade_timeout: number     // 秒
  },
  players: [PlayerState],
  current_round: number,
  current_turn: number,       // 当前报价者在本轮中的序号
  created_at: timestamp
}
```

### 7.3 玩家状态（PlayerState）
```
{
  user_id: string,
  seat_number: number,        // 座位号 1-n
  net_position: number,       // 净仓位
  cash: number,               // 累计现金流
  is_ready: boolean,
  is_connected: boolean
}
```

### 7.4 报价记录（Quote）
```
{
  id: string,
  game_id: string,
  round: number,
  turn: number,
  quoter_id: string,          // 报价者
  bid: number,
  ask: number,
  spread: number,
  bid_filled: boolean,        // Bid侧是否被成交
  ask_filled: boolean,        // Ask侧是否被成交
  bid_taker_id: string?,      // Bid侧成交对手
  ask_taker_id: string?,      // Ask侧成交对手
  created_at: timestamp
}
```

### 7.5 成交记录（Trade）
```
{
  id: string,
  game_id: string,
  quote_id: string,           // 关联报价
  round: number,
  buyer_id: string,
  seller_id: string,
  price: number,
  side: enum,                 // bid_side | ask_side
  created_at: timestamp
}
```

### 7.6 信息/线索（Clue）
```
{
  id: string,
  game_id: string,
  round: number,
  content: string,            // 线索内容
  created_at: timestamp
}
```

## 8. 技术方案建议

### 8.1 技术栈

| 层级 | 推荐方案 | 说明 |
|------|---------|------|
| 前端 | React + TypeScript + Tailwind CSS | 组件化开发，响应式设计 |
| 实时通信 | WebSocket（Socket.IO） | 报价/成交需毫秒级实时推送 |

## 9. 边界情况与异常处理

| 场景 | 处理方式 |
|------|---------|
| 报价超时 | 自动跳过，视为放弃，不影响价差递减（下一位价差要求不变） |
| 成交窗口超时 | 自动关闭，标记为未成交 |
| 玩家断线 | 保留座位 30 秒，超时自动托管（不报价、不成交），重连后恢复 |
| 所有玩家均达到持仓上限 | 本轮提前结束，进入下一轮（或直接结算） |
| 价差已缩小到 tick_size 且仍无成交 | 后续报价者维持 tick_size 价差 |
| 同时点击买入 | 服务端以接收时间戳判定优先，仅第一个生效 |
| 玩家尝试超过仓位限制的交易 | 服务端拒绝，返回错误提示 |
| 报价不合法（非 tick_size 整数倍、价差超限等） | 前端实时校验 + 后端二次校验，拒绝不合法报价 |
| 管理员断线 | 游戏暂停，等待管理员重连 |

## 10. UI/UX 设计要点

### 10.1 设计风格
- 深色主题（金融交易风格），辅以亮色高亮关键数据
- 信息密度适中，核心操作一步到位
- 移动端友好（响应式布局）

### 10.2 关键交互
- **报价输入**：支持滑块 + 数字输入双模式，实时显示价差和合法性
- **成交按钮**：大按钮、醒目颜色（买入绿色、卖出红色），点击后立即反馈
- **倒计时**：环形进度条 + 数字，最后 5 秒闪烁警告
- **仓位可视化**：横向仓位条，正中间为 0，左侧空头右侧多头，实时动画
- **成交动画**：成交时有简短的视觉 + 音效反馈
- **排行榜动画**：排名变化时有滑动动画

### 10.3 页面结构
```
┌──────────────────────────────────────────────┐
│  顶部导航栏：房间号 | 轮次 | 倒计时 | 设置     │
├──────────┬───────────────────┬───────────────┤
│          │                   │               │
│  信息/    │    报价板          │   排行榜       │
│  线索区   │    (Order Book)   │   (实时PnL)   │
│          │                   │               │
│          ├───────────────────┤               │
│          │                   │               │
│  交易     │    操作区          │   个人仓位     │

```

## 11. 非功能性需求

| 需求 | 指标 |
|------|------|
| 并发支持 | 单房间 10 人同时在线 |
| 延迟 | 成交操作端到端延迟 < 200ms |
| 可用性 | 支持玩家断线重连 |
| 数据一致性 | 所有仓位和交易数据强一致（服务端权威） |
| 安全性 | 真实值 V 仅存于服务端，客户端无法获取 |
| 可扩展性 | 支持同时运行多个房间 |
